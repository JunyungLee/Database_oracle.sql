 Oracle :: 그룹함수 

 

 

 

 

그룹별로 묶어 통계자료를 만드는데 사용되는 함수들을 다루어 볼 것이다.  

 

sum            -   합

avg            - 평균

count         - 총 계수

max           - 최대값

min            - 최소값

stddev        - 표준편차

variance      - 분산

 

 

SQL> select sum(sal),avg(sal),min(sal),count(sal)
  2  from emp;

  SUM(SAL)   AVG(SAL)   MIN(SAL) COUNT(SAL)
---------- ---------- ---------- ----------
     29025 2073.21429        800         14

SQL> select ename, sal
  2  from emp
  3  where sal>avg(sal);
where sal>avg(sal)
          *
ERROR at line 3:
ORA-00934: group function is not allowed here

그룹 함수는 where 절에 적용할 수 없다. where 절은 단일행으로 조건을 검색하기 때문이다.

 

SQL> select ename, avg(sal)
  2  from emp;
select ename, avg(sal)
       *
ERROR at line 1:
ORA-00937: not a single-group group function

 또한 ename 과 같이 일반 select절에 사용된 컬럼과 그룹함수는 함께 사용할 수 없다. 그래서 사용되는 함수가

 

GROUP BY 절이다. 195p

 HAVING 절  

  

 

그룹함수를 적용한 컬럼이 조건으로 오는 절.

 

직업별 평균급여를 구하는 예를 생각해보자. sal이 2000인 이상인 경우에만 직업별로 묶어 평균을 구하고 그 평균값이 3000이상인 데이터만 출력하는 것을 예로 보자.

 

SQL> select job, avg(sal)
  2  from emp
  3  where sal>=2000
  4  group by job
  5  having avg(sal)>=3000;

​

JOB                  AVG(SAL)
------------------ ----------
PRESIDENT                5000
ANALYST                  3000

 

예제

 

1. 직업별 인원수

2. 부서별 인원수를 구하세요. 단, 급여가 2000이상인 데이터를 이용하여 작업

3. 부서별 인원수를 구하세요. 단, 인원이 3명 이상인 부서만 출력

4. 직업별 인원수를 구하세요. 단, 직업별 인원수가 3명 이상인 데이터만 출력. 도한 SALESMAN은 제외하고 인원수를 구하세요

문제 1. 각 부서별로 월급의 평균을 조회한 후 평균값이 6000 보다 큰 값만 출력하고 부서번호가 없을 경우는 'No Department'로 출력되도록 하시오. 출력되는 부서별 평균값은 소수점 일의 자리에서 반올림을 시킵니다. (예 : 4.5 인 경우 5로 변함)
